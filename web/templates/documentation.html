Web Interface Guide

Overview

The ML Project Framework web interface provides an interactive way to:

  • Upload and preview datasets
  • Configure machine learning pipelines
  • Train models with custom parameters
  • View and compare experiment results

Getting Started

  1. Start the Web Server: python run_web.py
  2. Open Browser: Navigate to http://localhost:5000
  3. Upload Data: Go to Pipeline tab and upload your CSV, Excel, or Parquet file
  4. Configure: Set your training parameters
  5. Train: Click "Train Model" button
  6. Review: Check results in the Results tab

Features

Dashboard

View system overview including:

  • Total experiments completed
  • Last training accuracy
  • Recent experiments list
  • Quick start guide

Pipeline

Complete training workflow in one place:

  • Upload Data: Drag and drop or click to upload
  • Data Preview: See data shape, columns, and sample rows
  • Configuration: Choose problem type, algorithm, and parameters
  • Training: Train model with one click
  • Results: View training metrics immediately

Results

Experiment tracking and analysis:

  • Browse all completed experiments
  • View detailed metrics for each experiment
  • Compare confusion matrices and classification reports
  • Track model performance over time

Configuration Options

Problem Type

  • Classification: For categorical target variables
  • Regression: For continuous target variables

Algorithms

  • Random Forest: Ensemble method, good general-purpose choice
  • Gradient Boosting: Powerful boosting algorithm
  • Logistic Regression: Linear model for classification
  • Support Vector Machine: Effective for high-dimensional data
  • Linear Regression: Linear model for regression

Feature Scaling

  • Standard Scaler: Mean 0, std 1 (best for normal distributions)
  • MinMax Scaler: Scale to [0, 1] range
  • Robust Scaler: Uses median and IQR (resistant to outliers)

Missing Values Strategy

  • Drop: Remove rows with missing values
  • Mean: Fill with column mean
  • Median: Fill with column median

Model Parameters

Advanced users can provide custom hyperparameters as JSON:

{"n_estimators": 200, "max_depth": 15, "min_samples_split": 5}

Tips & Best Practices

Data Preparation

  • Ensure your CSV/Excel has a column named "target" (or specify a different name)
  • Remove or handle missing values appropriately
  • Encode categorical features before upload (or use another preprocessing tool)

Model Training

  • Start with default parameters to establish a baseline
  • Use a reasonable test size (0.2 means 80-20 split)
  • For high-dimensional data, consider removing outliers
  • Experiment with different algorithms to find the best fit

Results Interpretation

  • Accuracy: Overall correct predictions (watch for imbalanced data)
  • F1 Score: Harmonic mean of precision and recall
  • ROC-AUC: Area under the ROC curve (0.5 is random, 1.0 is perfect)
  • Confusion Matrix: Shows true positives, false positives, etc.

Troubleshooting

Upload Fails

  • Check file format (CSV, Excel, or Parquet only)
  • Ensure file size is under 16 MB
  • Verify file is not corrupted

Training Fails

  • Verify target column name matches your data
  • Check that target column exists in the data
  • Ensure JSON is valid if using custom model parameters
  • Check logs in logs/ directory for details

Slow Training

  • Try reducing data size or number of features
  • Use simpler models (e.g., Logistic Regression)
  • Reduce number of model estimators

API Endpoints

For advanced users, the web interface also provides REST API endpoints:

File Operations

  • POST /api/upload - Upload dataset
  • POST /api/preview-data - Preview dataset

Training & Models

  • POST /api/train-model - Train a model
  • GET /api/algorithms - Get available algorithms

Experiments

  • GET /api/experiments - List all experiments
  • GET /api/experiment/{filename} - Get experiment details

Configuration

  • GET /api/config - Get system configuration
  • GET /api/status - Get system status

FAQ

Q: Can I use my own data?

A: Yes! Upload any CSV, Excel, or Parquet file. Make sure it has a target column.

Q: How do I know which algorithm to use?

A: Random Forest is a good starting point for most problems. Try different algorithms and compare results.

Q: Can I download trained models?

A: Currently, models are saved to the models/ directory. Models are stored as pickle files.

Q: What's the maximum file size?

A: 16 MB for uploaded files. For larger datasets, consider splitting the data.

Q: Can I use this in production?

A: This is a development interface. For production, deploy with proper security and use the Python API directly.
